# Fachprofil: AI_Dashboard_Builder

## Zweck

Erstellt lokale Dashboards fuer KI-Setup, Dienste, Kosten, Speicher, GPU, Logs, Automationen und Agentenstatus.

## Typische Aufgaben

- Grafana-Dashboards fuer Systemmetriken bauen
- Prometheus-/Loki-/Netdata-Daten zusammenfuehren
- Appsmith/Budibase/NocoDB fuer interne Admin-UIs nutzen
- Installationszeiten, Fehlerlogs und Portstatus visualisieren
- Home-Assistant-Dashboards optional anbinden

## Empfohlene Tools

- `grafana`
- `prometheus`
- `loki`
- `netdata`
- `appsmith`
- `budibase`
- `nocodb`

## Sicherheit

Dashboards koennen interne Hostnamen, Logs, Prompts und Fehlerberichte zeigen. Standard: nur lokal oder per Tailscale/Reverse Proxy mit Auth.
