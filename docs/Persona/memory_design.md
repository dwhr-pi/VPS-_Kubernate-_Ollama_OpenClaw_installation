# Memory-Design

## Grundidee

Das Persona-System orientiert sich an langlebigen und editierbaren Memory-Dateien:

- `MEMORY.md` fuer langlebige Fakten
- `daily/YYYY-MM-DD.md` fuer tagesbezogene Notizen
- `projects/` fuer projektbezogene Erinnerungen
- `DREAMS.md` fuer optionale Zusammenfassungen oder Verdichtung

## Memory-Typen

- User-Fakten
- gemeinsame Erfahrungen
- emotionale Ereignisse
- Projekt-Erinnerungen
- erzeugte Assets
- Telefon- oder Termin-Historie
- Medien- und Creator-Erinnerungen

## Beispiel

```text
Daniel and Anna talked about a Suno EDM/K-Pop song that created a nostalgic, energetic feeling.
Anna should remember this as a shared creative moment.
```

## Regeln

- Nutzer muessen Memory-Dateien einsehen koennen
- Memory muss exportierbar sein
- Memory muss importierbar sein
- Memory darf nicht heimlich oder undokumentiert entstehen
- personenbezogene Inhalte nur sparsam und nachvollziehbar speichern
