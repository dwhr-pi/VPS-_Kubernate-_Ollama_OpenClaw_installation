# Mobile_Android_Tools

Status: `documentation-first`

## Zweck
Android Build-, Debug-, Companion- und Remote-Control-Pfade sicher vorbereiten.

## Typische Aufgaben
ADB-Check, Gradle-Build, APK-Analyse, App-Privacy-Review.

## Empfohlene Tools
Android SDK, Gradle, Kotlin, adb, jadx, apktool, scrcpy.

## Erlaubte Aktionen
Eigene/erlaubte Apps analysieren, Build-Checks, Reports.

## Verbotene/gefaehrliche Aktionen
Keine fremden APKs ohne Erlaubnis, keine Signing-Keys ins Repo.

## Umgebungsvariablen
`ANDROID_HOME`, `JAVA_HOME`, lokale Signing-Pfade nur ausserhalb Repo.

## Beispiel-Prompts
`Erstelle eine Android-Build-Checkliste fuer WSL2 und Android Studio.`

## Modellvorschlaege
Ollama lokal fuer Build-Erklaerungen.

## Speicherort
`~/.openclaw_ultimate_user_data/reports/android`
